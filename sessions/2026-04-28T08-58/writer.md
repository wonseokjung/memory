### 3. 핵심 설계: 3단계 AI 자동화 파이프라인 아키텍처 (1:30 – 4:00)

**A. Phase 1: 목표 정의 및 입력 설계 (The Input Layer)**
*   **설명:** 시스템을 시작하기 전에, **무엇을 자동화할지(목표)**와 **어떤 데이터가 필요한지(입력 변수)**를 명확히 정의하는 단계입니다.
*   **핵심 행동:** 'Input'의 구조화 (예: 목표 설정 $\rightarrow$ 필요 정보 목록 $\rightarrow$ 출력 형식 정의).

**B. Phase 2: 프로세스 통합 및 로직 설계 (The Integration Layer)**
*   **설명:** 가장 중요한 단계. 서로 다른 AI/툴을 어떻게 연결할지 **논리적 흐름(Logic Flow)**을 설계합니다. 여기서는 Zapier, Make, API 등을 사용하여 데이터가 한 툴에서 다음 툴로 오류 없이 전달되도록 연결하는 방법을 설명합니다.
*   **핵심 행동:** 'Connection Point' 설정. 각 단계의 결과물이 다음 단계의 입력이 되도록 프롬프트와 API 호출을 정의하는 방법 제시.

**C. Phase 3: 검증, 루프 및 배포 (The Output & Loop Layer)**
*   **설명:** 시스템이 실제로 작동하는지 **테스트(Debugging)**하고, 결과가 만족스러운지 확인한 후 다시 피드백을 주는 **반복 루프(Loop)**를 설계합니다. 자동화는 한 번 만들고 끝내는 것이 아니라, 지속적으로 개선하는 것입니다.
*   **핵심 행동:** 오류 발생 시의 예외 처리(Error Handling) 로직 삽입 방법 설명.
