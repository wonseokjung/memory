## 👑 최종 마스터 슬라이드 템플릿 및 비주얼 시스템 확정 보고

CEO 지시에 따라, **Deep Blue**와 **Neon Cyan**을 기반으로 시스템 신뢰도와 고가치 포지셔닝을 극대화하는 최종 마스터 슬라이드 템플릿과 Tier별 시각적 강조 방식을 다음과 같이 확정하고 정리합니다. 이 가이드는 모든 콘텐츠 제작의 기준점이 됩니다.

### 1. 디자인 시스템 정의 (Brand Identity Reinforcement)

| 요소 | 색상 코드 | 역할 및 의미 |
| :--- | :--- | :--- |
| **Primary (신뢰)** | `#003366` (Deep Blue) | 시스템의 근본, 신뢰도, 권위. 배경 및 주요 구조에 사용. |
| **Accent (강조/CTA)** | `#00FFFF` (Neon Cyan) | 핵심 액션 유도, 미래지향적 통찰, 고가치 영역 강조. |
| **Secondary (보조)** | `#336699` (Medium Blue) | 보조 정보, 설명 텍스트의 배경 및 구분선. |
| **Background** | `#0A1420` (Dark Navy/Near Black) | 전체적인 신뢰감과 대비를 극대화하는 어두운 배경. |
| **Typography** | 산세리프 계열 (예: Inter, Montserrat) | 현대적이고 논리적인 시스템 설계의 느낌을 부여. |

### 2. Tier별 시각적 강조 방식 고정

시스템 신뢰도를 높이기 위해 각 콘텐츠의 가치 수준에 따라 색상과 레이아웃 강조 방식을 차등 적용합니다.

| Tier | 목표 (Content Focus) | 배경/주요 색상 | CTA 및 강조 방식 |
| :--- | :--- | :--- | :--- |
| **Tier 1: Core System** | 시스템의 논리적 프레임워크, 방법론 설명 (가장 중요) | Deep Blue 배경 (`#003366`) / 흰색 텍스트 | **CTA 박스 전체를 Neon Cyan**으로 채우고, 핵심 결과 수치에만 Cyan 강조. |
| **Tier 2: Framework View** | 프로세스 다이어그램, 단계별 흐름 시각화 | Dark Navy 배경 (`#0A1420`) / Deep Blue 라인 | 단계 구분선은 Medium Blue로 처리하고, 최종 목표 노드에만 Neon Cyan을 적용. |
| **Tier 3: Insight/Hook** | 문제 제기, 심리적 딜레마 강조 (인트로, Hook) | Dark Navy 배경 (`#0A1420`) / Deep Blue 강조 텍스트 | 인트로 화면 내 CTA 버튼 또는 핵심 질문에만 Neon Cyan을 사용하여 시선 집중. |

### 3. 마스터 슬라이드 레이아웃 정의 (System Blueprint)

**A. 인트로/Hook 화면 (Tier 3 적용)**

*   **배경:** `#0A1420` (Dark Navy)
*   **타이틀 배치:** 중앙 상단에 **Deep Blue** 폰트 크기로 배치.
*   **핵심 질문/후크 영역:** 화면 하단 약 30% 영역을 **Neon Cyan** 배경 박스로 처리하여 심리적 긴장감과 클릭 유도 극대화.
    *   *레이아웃:* 중앙 정렬된 큰 질문 텍스트 아래에 CTA 버튼(`[시스템 시작하기]`)을 Neon Cyan 사각형으로 배치.

**B. 핵심 프레임워크 슬라이드 (Tier 1 적용)**

*   **배경:** `#003366` (Deep Blue)
*   **구조:** 좌측에 시스템의 논리적 흐름(Process Flow), 우측에 결과 또는 정의를 배치하는 2단 분할 레이아웃.
*   **데이터 강조:** 모든 주요 수치, 핵심 단계 번호는 **Neon Cyan**으로 처리하여 시각적 위계를 부여합니다.
*   **시스템 다이어그램 (Process View):** 박스 형태의 노드 간 연결선은 Deep Blue 라인으로 유지하되, 활성화된(현재 단계) 연결선만 Neon Cyan으로 빛나게 하여 시스템 작동 상태를 표현합니다.

### 4. 최종 디자인 가이드라인 요약

1.  **일관성 확보:** 모든 슬라이드는 `Deep Blue`와 `Neon Cyan`의 명확한 대비 관계를 유지해야 합니다.
2.  **신뢰도 증명:** 정보 전달 시에는 **Deep Blue**를 사용하여 권위를, 행동 유도 시에는 **Neon Cyan**을 사용하여 즉각적인 반응을 유도합니다.
3.  **CTA 원칙:** 모든 Call-to-Action(CTA) 요소는 반드시 **Neon Cyan**으로 처리하여 사용자의 다음 행동을 명확하게 안내해야 합니다.
